Entity #

继承自

GraphNode

实体是 PlayCanvas 游戏的核心基本体。一般来说，游戏中的对象将由一个实体和一组组件组成，这些组件由各自的组件系统管理。其中一个组件可能是 ScriptComponent，它允许您编写自定义代码以附加到实体。

实体唯一标识对象，并为其从 GraphNode 继承的位置和方向提供变换，以便可以添加到场景图形中。Component 和 ComponentSystem 提供给实体特定类型行为的逻辑。e、 g.渲染模型或播放声音的能力。组件特定于实体的一个实例，并被附加（例如 this.Entity.model）ComponentSystems 允许访问所有实体和组件，并且被附加到 AppBase。

🏰 构造器 #

Entity([name] : string, [app] : AppBase) #

创建新实体。

javascript
var entity = new pc.Entity();

// 将组件添加到实体
entity.addComponent("camera", {
  fov: 45,
  nearClip: 1,
  farClip: 10000,
});

// 将实体添加到场景图中
app.root.addChild(entity);

// 移动实体
entity.translate(10, 0, 0);

// 或者通过直接设置其位置进行翻译
var p = entity.getPosition();
entity.setPosition(p.x + 10, p.y, p.z);

// 更改实体在局部空间中的旋转
var e = entity.getLocalEulerAngles();
entity.setLocalEulerAngles(e.x, e.y + 90, e.z);

// 或使用rotateLocal
entity.rotateLocal(0, 90, 0);

参数列表

  • name : 实体的非唯一名称，默认为“Untitled”。
  • app : 实体所属的应用程序，默认为当前应用程序。

🕹️ 方法 #

⭐ 共有方法请参照其基类：GraphNode

📌 addComponent(type : string, [data] : object) : Component, null
#

创建新组件并将其添加到实体中。使用此选项可以向实体添加功能，如渲染模型、播放声音等。

javascript
var entity = new pc.Entity();

// 添加具有默认特性的灯光组件
entity.addComponent("light");

// 添加具有某些指定特性的相机组件
entity.addComponent("camera", {
  fov: 45,
  clearColor: new pc.Color(1, 0, 0),
});

参数列表

  • type : 要添加的组件的名称。有效字符串为：
    • “anim” - 请参见 AnimComponent
    • “animation” -请参见 AnimationComponent
    • "audiolistener" - 请参见 AudioListenerComponent
    • "button" - 请参见 ButtonComponent
    • "camera" - 请参见 CameraComponent
    • "collision" - 请参见 CollisionComponent
    • "element" - 请参见 ElementComponent
    • "layoutchild" - 请参见 LayoutChildComponent
    • "layoutgroup" - 请参见 LayoutGroupComponent
    • "light" - 请参见 LightComponent
    • "model" - 请参见 ModelComponent
    • "particlesystem" - 请参见 ParticleSystemComponent
    • "render" - 请参见 RenderComponent
    • "rigidbody" - 请参见 RigidBodyComponent
    • "screen" - 请参见 ScreenComponent
    • "script" - 请参见 ScriptComponent
    • "scrollbar" - 请参见 ScrollbarComponent
    • "scrollview" - 请参见 ScrollViewComponent
    • "sound" - 请参见 SoundComponent
    • "sprite" - 请参见 SpriteComponent
  • data : 特定组件类型的初始化数据。有关此参数的有效值的详细信息，请参阅每个特定组件的 API 参考页。

返回值

附加到实体的新组件，如果有错误，则为空。


📌 clone() : this
#

创建实体的深副本。复制具有所有组件和子体的完整图元层次结构。注意，此实体不在层次结构中，必须手动添加。

javascript
var e = this.entity.clone();

// 将克隆添加为原始的亲缘关系
this.entity.parent.addChild(e);

返回值

一个新的实体，是原始实体的深拷贝。


📌 destroy()
#

从实体中删除所有零部件，并将其从实体层次结构中分离。然后递归地销毁所有祖先实体。

javascript
var firstChild = this.entity.children[0];
firstChild.destroy(); // 删除子级、所有组件并从层次结构中删除

📌 findByGuid(guid : string) : Entity, null
#

使用 GUID 查找此实体的后代。

参数列表

  • guid : 要搜索的 GUID。

返回值

具有匹配 GUID 的实体，如果找不到实体，则为空。


📌 findComponent(type : string) : Component
#

在实体及其所有子体中搜索指定类型的第一个组件。

javascript
// 获取层次树中以该实体开头的第一个找到的灯光组件
var light = entity.findComponent("light");

参数列表

  • type : 要检索的组件类型的名称。

返回值

指定类型的组件，如果实体或其任何后代有一个组件。否则返回未定义的。


📌 findComponents(type : string) : Component[]
#

在实体及其所有子体中搜索指定类型的所有组件。

javascript
// 获取层次树中以该实体开头的所有灯光组件
var lights = entity.findComponents("light");

参数列表

  • type : 要检索的组件类型的名称。

返回值

实体或其任何后代中指定类型的所有组件。如果找不到，则返回空数组。


📌 removeComponent(type : string)
#

从实体中删除组件。

javascript
var entity = new pc.Entity();
entity.addComponent("light"); // 添加新灯光组件

entity.removeComponent("light"); // 删除灯组件

参数列表

  • type : 组件类型的名称。

📲 事件 #

📌 destroy
#

实体销毁后激发。

javascript
entity.on("destroy", function (e) {
  console.log("entity " + e.name + " has been destroyed");
});

参数列表

  • entity : Entity : 被销毁的实体。
