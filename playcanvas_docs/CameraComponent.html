CameraComponent #

继承自

Component

“摄影机组件”允许实体渲染场景。场景需要渲染至少一个启用的摄影机组件。请注意，可以同时启用多个相机组件（例如，用于分割屏幕或屏幕外渲染）。

javascript
// 将pc.CameraComponent添加到实体
var entity = new pc.Entity();
entity.addComponent("camera", {
  nearClip: 1,
  farClip: 100,
  fov: 55,
});

// 获取实体上的pc.CameraComponent
var cameraComponent = entity.camera;

// 更新相机组件的特性
entity.camera.nearClip = 2;

🏰 构造器 #

CameraComponent(system : CameraComponentSystem, entity : Entity) #

创建新的 CameraComponent 实例。

参数列表

  • system : 创建此组件的组件系统。
  • entity : 此组件附加到的实体。

🔮 属性 #

⭐ 共有属性请参照其基类：Component

📌 aspectRatio : number
相机的纵横比（宽度除以高度）。如果 aspectRatioMode 为 ASPECT_AUTO，则每帧都会自动计算此值，并且您只能读取它。如果为 ASPECT_MANUAL，则可以设置此值。

📌 aspectRatioMode : number
相机的纵横比模式。可以是：

  • ASPECT_AUTO : 纵横比将根据当前渲染目标的宽度除以高度计算。
  • ASPECT_MANUAL : 使用 aspectRatio 值。

默认为 ASPECT_AUTO 。

📌 calculateProjection : CalculateMatrixCallback
您可以提供自定义功能来手动计算相机投影矩阵。可以用于复杂的效果，如做斜投影。使用组件的作用域调用函数。论据：

  • Mat4 transformMatrix：函数输出
  • view：视图的类型。可以是 VIEW_CENTER、VIEW_LEFT 或 VIEW_RIGHT。

左侧和右侧仅用于立体渲染。

📌 calculateTransform : CalculateMatrixCallback
您可以提供自定义功能来手动计算相机变换矩阵。可用于反射等复杂效果。使用组件的作用域调用函数。论据：

  • Mat4 transformMatrix：函数输出
  • view：视图的类型。可以是 VIEW_CENTER、VIEW_LEFT 或 VIEW_RIGHT。

左侧和右侧仅用于立体渲染。

📌 clearColor : Color
用于在摄影机开始渲染之前清除画布的颜色。默认为[0.75，0.75，0.75,1]。

📌 clearColorBuffer : boolean
如果为 true，相机将清除颜色缓冲区，使其颜色设置为 clearColor。默认为 true。

📌 clearDepthBuffer : boolean
如果为真，相机将清除深度缓冲区。默认为 true。

📌 clearStencilBuffer : boolean
如果为 true，相机将清除模板缓冲区。默认为 true。 果为真，相机将清除深度缓冲区。默认为 true。

📌 cullFaces : boolean
如果为真，相机将拍摄素材。将其考虑在内。否则，将渲染正面和背面。默认为 true。

📌 disablePostEffectsLayer : number
停止应用相机后处理的层的层 ID。默认为 LAYERID_UI，这将导致后处理不应用于 UI 层和相机的任何后续层。设置为未定义，以便将后期处理应用于相机的所有层。

📌 farClip : number
与摄影机之间的距离，在此距离之后将不会进行渲染。默认为 1000。

📌 flipFaces : boolean
如果为真，相机将反转正面和背面。对于反射渲染很有用。默认为 false。

📌 fov : number
相机的视野（以度为单位）。通常这是 Y 轴视野，请参见 CameraComponent#horizontalFov。仅用于 PROJECTION_PERSPECTIVE 相机。默认为 45。

📌 frustum : Frustum
查询相机的视锥形状。

📌 frustumCulling : boolean
控制针对摄影机平截体的网格实例剔除，即渲染时是否应忽略摄影机外部的对象。如果为 false，则无论可见性如何，场景中的所有网格实例都由摄影机渲染。默认为 false。

📌 horizontalFov : boolean
设置要用于视野计算的轴。默认为 false。

📌 layers : number[]
此摄影机应属于的层 id（ Layer#id ）数组。不要推、弹出、拼接或修改此阵列，如果要更改它，请设置一个新阵列。默认为[LAYERID_WORLD、LAYERID_DEPTH、LAYER ID_SKYBOX、LAYERID_UI、LAYERID_IMMEDIATE]。

📌 nearClip : number
与摄影机之间的距离，在此距离之前不会进行渲染。默认为 0.1。

📌 onPostRender : function
相机渲染场景后调用的自定义函数。

📌 onPreRender : function
在摄影机渲染场景之前调用的自定义函数。

📌 orthoHeight : number
正交视图窗口的半高（在 Y 轴上）。仅用于 PROJECTION_ORTHOGRAPHIC 相机。默认为 10。

📌 postEffectsEnabled : PostEffectQueue
此相机的后期效果队列。使用此选项可以在摄影机中添加或移除后期效果。

📌 priority : number
控制渲染摄影机的顺序。首先渲染优先级值较小的摄影机。默认为 0。

📌 projection : number
用于渲染摄影机的投影类型。可以是：

  • PROJECTION_PERSPECTIVE : 透视投影。相机平截头体类似于一个截断的金字塔。
  • PROJECTION_ORTHOGRAPHIC : 正交投影。相机视锥是长方体。

默认为 PROJECTION_PERSPECTIVE 。

📌 projectionMatrix : Mat4
查询相机的投影矩阵。

📌 rect : Vec4
控制相机在屏幕上以标准化屏幕坐标渲染的位置。默认为[0，0，1，1]。

📌 renderTarget : RenderTarget
渲染执行摄影机渲染的目标。如果未设置，它将仅渲染到屏幕。

📌 scissorRect : Vec4
剪裁不在矩形中的所有像素。值的顺序是[x，y，width，height]。默认为[0，0，1，1]。

📌 viewMatrix : Mat4
查询相机的视图矩阵。

🕹️ 方法 #

⭐ 共有方法请参照其基类：Component

📌 calculateAspectRatio([rt] : RenderTarget) : number
#

计算给定渲染目标的纵横比值。

参数列表

  • rt : 可选渲染目标。如果未指定，则使用后台缓冲区。

返回值

渲染目标（或后缓冲区）的纵横比。


📌 endXr([callback] : XrErrorCallback)
#

尝试结束此相机的 XR 会话。

javascript
// 在具有相机组件的图元上
this.entity.camera.endXr(function (err) {
  // XR中不再有
});

参数列表

  • callback : 会话结束后调用的可选回调函数。回调有一个参数错误-如果成功结束 XR 会话，则为空。

📌 requestSceneColorMap(enabled : boolean)
#

请求场景生成包含场景颜色贴图的纹理。注意，这个调用是累积的，对于每个启用请求，需要调用一个禁用请求。

参数列表

  • enabled : 如果为 True，则请求生成；如果为 false，则禁用生成。

📌 requestSceneDepthMap(enabled : boolean)
#

请求场景生成包含场景深度贴图的纹理。注意，这个调用是累积的，对于每个启用请求，需要调用一个禁用请求。

参数列表

  • enabled : 如果为 True，则请求生成；如果为 false，则禁用生成。

📌 screenToWorld(screenx : number, screeny : number, cameraz : number, [worldCoord] : Vec3) : Vec3
#

将点从二维屏幕空间转换为三维世界空间。

javascript
// 获取从屏幕单击位置发射的3D射线的起点和终点
var start = entity.camera.screenToWorld(clickX, clickY, entity.camera.nearClip);
var end = entity.camera.screenToWorld(clickX, clickY, entity.camera.farClip);

// 使用光线坐标执行光线投射
app.systems.rigidbody.raycastFirst(start, end, function (result) {
  console.log("Entity " + result.entity.name + " was selected");
});

参数列表

  • screenx : PlayCanvas 画布元素上的 X 坐标。应在 0 到画布的范围内。应用程序画布元素的 offsetWidth。
  • screeny : PlayCanvas 画布元素上的 Y 坐标。应在 0 到画布的范围内。应用程序画布元素的 offsetHeight。
  • cameraz : 在世界空间中创建新点与相机的距离。
  • worldCoord : 用于接收世界坐标结果的三维矢量。

返回值

世界空间坐标。


📌 startXr(type : string, spaceType : string, [options] : object)
#

尝试使用此相机启动 XR 会话。

javascript
// 在具有相机组件的图元上
this.entity.camera.startXr(pc.XRTYPE_VR, pc.XRSPACE_LOCAL, {
  callback: function (err) {
    if (err) {
      // failed to start XR session
    } else {
      // in XR
    }
  },
});

参数列表

  • type : 会话的类型。可以是以下之一：XRTYPE_INLINE：内联-始终可用的会话类型。它的功能可用性有限，并呈现为 HTML 元素。 / XRTYPE_VR：沉浸式 VR 会话，提供对 VR 设备的独占访问，具有最佳可用跟踪功能。 / XRTYPE_AR：沉浸式 AR 会话，提供对 VR/AR 设备的独占访问，该设备旨在与真实环境相融合。
  • spaceType : 引用空间类型。可以是以下之一：XRSPACE_VIEWER:查看器-始终支持空间，具有一些基本的跟踪功能。 / XRSPACE_LOCAL:LOCAL-表示创建时查看器附近具有本地原点的跟踪空间。它适用于就座或基本的本地 XR 会话。 / XRSPACE_LOCALFLOOR:Local Floor-表示一个跟踪空间，该空间在地板上的本地原点位于用户可以站立的安全位置。y 轴在地板标高处等于 0。底层值可能由底层平台估算。它适用于就座或基本的本地 XR 会话。 / XRSPACE_BOUNDEDFLOOR:Bounded Floor-表示一个跟踪空间，其原始原点位于地板上，预计用户将在预先确定的边界内移动。 / XRSPACE_UNBOUNDED:UNBOUNDED-表示一个跟踪空间，在该空间中，用户可以在其环境中自由移动，可能距离起点很远。
  • options : 对象，其中包含 XR 会话初始化选项。
    • options.optionalFeatures : string[] : XRSession 启动的可选功能。它用于访问其他 WebXR 规范扩展。
    • options.imageTracking : boolean : 设置为 true 以尝试启用 XrImageTrack。
    • options.planeDetection : boolean : 设置为 true 以尝试启用 XrPlaneDetection。
    • options.callback : XrErrorCallback : 会话启动后调用的可选回调函数。回调有一个参数 Error-如果 XR 会话成功启动，则该参数为 null。
    • options.depthSensing : object : 具有深度感应参数的可选对象，用于尝试启用 XrDepthSensing。
      • options.depthSensing.usagePreference : string : 深度传感的可选使用首选项，可以是“cpu 优化”或“gpu 优化”（XRDEPTHSENSINGUSAGE_*），默认为“cpu 最优化”。最受青睐和支持的将由底层深度传感系统选择。
      • options.depthSensing.dataFormatPreference : string : 深度传感的可选数据格式首选项。可以是“亮度 alpha”或“float32”（XRDEPTHSENSINGFORMAT_*），默认为“亮度 alpha”。最受青睐和支持的将由底层深度传感系统选择。

📌 worldToScreen(worldCoord : Vec3, [screenCoord] : Vec3) : Vec3
