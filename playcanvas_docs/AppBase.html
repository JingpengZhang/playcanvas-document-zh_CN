AppBase #

继承自

EventHandler

应用程序代表并管理您的 PlayCanvas 应用程序。如果您正在使用 PlayCanvas 编辑器进行开发，则会为您创建应用程序。您可以在脚本中访问 Application 实例。下面是一个框架脚本，它显示了如何访问初始化和更新函数中的应用程序“app”属性：

javascript
// 编辑器示例：从脚本访问 pc.Application
var MyScript = pc.createScript("myScript");

MyScript.prototype.initialize = function () {
  // Every script instance has a property 'this.app' accessible in the initialize...
  var app = this.app;
};

MyScript.prototype.update = function (dt) {
  // ...and update functions.
  var app = this.app;
};

如果使用引擎而不使用编辑器，则必须手动创建应用程序实例。

javascript
// 仅引擎示例：手动创建应用程序
var options = new AppOptions();
var app = new pc.AppBase(canvas);
app.init(options);

// 启动应用程序的主循环
app.start();

🎈 静态方法 #

📌 getApplication([id] : string) : AppBase, undefined
#

获取当前应用程序。在有多个正在运行的应用程序的情况下，该函数可以根据提供的画布 id 获取应用程序。当当前应用程序不可用时，该函数特别有用。例如，在浏览器开发工具的 JavaScript 控制台中。

参数列表

  • id : 如果已定义，则返回的应用程序应使用具有此 id 的画布。否则，将返回当前应用程序。

返回值

正在运行的应用程序（如果有）。


🔮 属性 #

📌 _batcher : BatchManager
应用程序的批处理管理器。

📌 assets : AssetRegistry
由应用程序管理的资产注册表。

javascript
// 在资产注册表中搜索标记为“vehicle”的所有资产
var vehicleAssets = this.app.assets.findByTag("vehicle");

📌 autoRender : boolean
如果为 true，则在每一帧调用应用程序的渲染函数。将 autoRender 设置为 false 对于渲染图像可能随时间推移而经常保持不变的应用程序非常有用。这可以大大减少应用程序在 CPU 和 GPU 上的负载。默认为 true。

javascript
// 禁用渲染每帧并仅在关键帧事件上渲染
this.app.autoRender = false;
this.app.keyboard.on(
  "keydown",
  function (event) {
    this.app.renderNextFrame = true;
  },
  this
);

📌 batcher : BatchManager
应用程序的批处理管理器。批处理管理器用于合并场景中的网格实例，从而减少绘制调用的总数，从而提高性能。

📌 elementInput : ElementInput
用于处理 ElementComponents 的输入。

📌 fillMode : string
画布的当前填充模式。可以是：

  • FILLMODE_NONE : 画布将始终与提供的大小匹配。
  • FILLMODE_FILL_WINDOW : 画布将简单地填充窗口，更改纵横比。
  • FILLMODE_KEEP_ASPECT : 画布将增长到尽可能填充窗口，同时保持纵横比。

📌 gamepads : GamePads
用于访问 GamePad 输入。

📌 graphicsDevice : GraphicsDevice
应用程序使用的图形设备。

📌 i18n : I18n
处理本地化。

📌 keyboard : Keyboard
键盘设备。

📌 lightmapper : Lightmapper
运行时 lightmapper。

📌 loader : ResourceLoader
资源加载器。

📌 maxDeltaTime : number
将每帧增量时间钳制到上限。很有用，因为从选项卡停用状态返回可以为 dt 生成巨大的值，这会对游戏状态产生不利影响。默认为 0.1（秒）。

javascript
// 不要夹住200ms或更短的帧间时间
this.app.maxDeltaTime = 0.2;

📌 mouse : Mouse
鼠标设备。

📌 renderNextFrame : boolean
设置为 true 以在主循环的下一次迭代中渲染场景。这仅在 AppBase#autoRender 设置为 false 时有效。渲染场景后，renderNextFrame 的值将再次设置为 false。

javascript
// 仅在按下空格键时渲染场景
if (this.app.keyboard.isPressed(pc.KEY_SPACE)) {
  this.app.renderNextFrame = true;
}

📌 resolutionMode : string
画布的当前分辨率模式可以是：

  • RESOLUTION_AUTO : 如果未提供宽度和高度，画布将调整大小以匹配画布客户端大小。
  • RESOLUTION_FIXED : 画布的分辨率将被固定。

📌 root : Entity
应用程序的根实体。

javascript
// 在场景层次的深度优先搜索中返回名为“Camera”的第一个实体
var camera = this.app.root.findByName("Camera");

📌 scene : Scene
由应用程序管理的场景。

javascript
// 设置应用程序场景的色调映射属性
this.app.scene.toneMapping = pc.TONEMAP_FILMIC;

📌 scenes : SceneRegistry
由应用程序管理的场景注册表。

javascript
// 在场景注册表中搜索名为“racetrack1”的项目
var sceneItem = this.app.scenes.find("racetrack1");

// 使用项目的url加载场景
this.app.scenes.loadScene(sceneItem.url);

📌 scripts : ScriptRegistry
应用程序的脚本注册表。

📌 systems : ComponentSystemRegistry
应用程序的组件系统注册表。应用程序构造函数将以下组件系统添加到其组件系统注册表中：

  • anim (AnimComponentSystem)
  • animation (AnimationComponentSystem)
  • audiolistener (AudioListenerComponentSystem)
  • button (ButtonComponentSystem)
  • camera (CameraComponentSystem)
  • collision (CollisionComponentSystem)
  • element (ElementComponentSystem)
  • layoutchild (LayoutChildComponentSystem)
  • layoutgroup (LayoutGroupComponentSystem)
  • light (LightComponentSystem)
  • model (ModelComponentSystem)
  • particlesystem (ParticleSystemComponentSystem)
  • rigidbody (RigidBodyComponentSystem)
  • render (RenderComponentSystem)
  • screen (ScreenComponentSystem)
  • script (ScriptComponentSystem)
  • scrollbar (ScrollbarComponentSystem)
  • scrollview (ScrollViewComponentSystem)
  • sound (SoundComponentSystem)
  • sprite (SpriteComponentSystem)
javascript
// 将全局重力设置为零
this.app.systems.rigidbody.gravity.set(0, 0, 0);
javascript
// 将全局音量设置为50%
this.app.systems.sound.volume = 0.5;

📌 timeScale : number
缩放全局时间增量。默认为 1。

javascript
// 将应用程序设置为半速运行
this.app.timeScale = 0.5;

📌 touch : TouchDevice
用于获取触摸事件输入。

📌 xr : XrManager
提供启动 VR/AR 会话能力的 XR 管理器。

javascript
// 检查VR是否可用
if (app.xr.isAvailable(pc.XRTYPE_VR)) {
  // VR可用
}

🕹️ 方法 #

⭐ 共有方法请参照其基类：EventHandler

📌 applySceneSettings(settings : object)
#

将场景设置应用于当前场景。当从非 URL 源解析或生成场景设置时非常有用。

javascript
var settings = {
  physics: {
    gravity: [0, -9.8, 0],
  },
  render: {
    fog_end: 1000,
    tonemapping: 0,
    skybox: null,
    fog_density: 0.01,
    gamma_correction: 1,
    exposure: 1,
    fog_start: 1,
    global_ambient: [0, 0, 0],
    skyboxIntensity: 1,
    skyboxRotation: [0, 0, 0],
    fog_color: [0, 0, 0],
    lightmapMode: 1,
    fog: "none",
    lightmapMaxResolution: 2048,
    skyboxMip: 2,
    lightmapSizeMultiplier: 16,
  },
};
app.applySceneSettings(settings);

参数列表

  • settings : 要应用的场景设置。
    • settings.physics : object : 要应用的物理设置。
      • settings.physics.gravity : number[] : 在物理模拟中表示全球重力的世界空间矢量。必须是具有三个数字元素的固定大小数组，对应于每个轴[X，Y，Z]。
    • settings.render : object : 要应用的渲染设置。
      • settings.render.global_ambient : number[] : 场景环境光的颜色。必须是具有三个数字元素的固定大小数组，对应于每个颜色通道[R，G，B]。
      • settings.render.fog : string : 场景使用的雾类型。可以是：FOG_NONE / FOG_LINEAR / FOG_EXP / FOG_EXP2 。
      • settings.render.fog_color : number[] : 雾的颜色（如果启用）。必须是具有三个数字元素的固定大小数组，对应于每个颜色通道[R，G，B]。
      • settings.render.fog_density : number : 雾的密度（如果启用）。仅当雾特性设置为 fog_EXP 或 fog_EXP2 时，此特性才有效。
      • settings.render.fog_start : number : 从线性雾开始的视点的距离。仅当雾特性设置为 fog_LINEAR 时，此特性才有效。
      • settings.render.fog_end : number : 距离线性雾达到最大值的视点的距离。仅当雾特性设置为 fog_LINEAR 时，此特性才有效。
      • settings.render.gamma_correction : number : 渲染场景时要应用的 gamma 校正。可以是：GAMMA_NONE / GAMMA_SRGB。
      • settings.render.tonemapping : number : 将片段写入帧缓冲区时要应用的色调映射变换。可以是：TONEMAP_LINEAR / TONEMAP_FILMIC / TONEMAP_HEJL / TONEMAP_ACES 。
      • settings.render.exposure : number : 曝光值会调整场景的整体亮度。
      • settings.render.skybox : number,null : 要用作场景天空盒的立方体贴图纹理的资源 ID。默认为 null。
      • settings.render.skyboxIntensity : number : 天空盒强度的倍增。
      • settings.render.skyboxMip : number : 要显示的天空盒的 mip 级别。仅对预过滤立方体贴图天空盒有效。
      • settings.render.skyboxRotation : number[] : 天空盒旋转。
      • settings.render.lightmapSizeMultiplier : number : 光照贴图分辨率倍增。
      • settings.render.lightmapMaxResolution : number : 最大光照贴图分辨率。
      • settings.render.lightmapMode : number : 光照贴图烘焙模式。可以是：BAKE_COLOR : 单色光照贴图 / BAKE_COLORDIR : 单色光照贴图+主光照方向（用于凹凸/镜面反射）。
      • settings.render.ambientBake : boolean : 启用将环境光烘焙到光照贴图中。
      • settings.render.ambientBakeNumSamples : number : 烘焙环境光时要使用的采样数。
      • settings.render.ambientBakeSpherePart : number : 烘焙环境光时要包含多少球体。
      • settings.render.ambientBakeOcclusionBrightness : number : 烘焙环境光遮挡的亮度。
      • settings.render.ambientBakeOcclusionContrast : number : 烘焙环境光遮挡的对比度。
      • settings.render.clusteredLightingEnabled : boolean : 启用群集照明。
      • settings.render.lightingShadowsEnabled : boolean : 如果设置为 true，簇光源将支持阴影。
      • settings.render.lightingCookiesEnabled : boolean : 如果设置为 true，则聚集照明将支持 cookie 纹理。
      • settings.render.lightingAreaLightsEnabled : boolean : 如果设置为 true，簇光源将支持区域光。
      • settings.render.lightingShadowAtlasResolution : number : 存储所有非方向阴影纹理的贴图集纹理的分辨率。
      • settings.render.lightingCookieAtlasResolution : number : 存储所有非定向 cookie 纹理的贴图集纹理的分辨率。
      • settings.render.lightingMaxLightsPerCell : number : 一个单元可以存储的最大灯光数。
      • settings.render.lightingShadowType : number : 所有阴影使用的阴影过滤类型。可以是：SHADOW_PCF1 : PCF 1x1 采样。/ SHADOW_PCF3 : PCF 3x3 采样。/ SHADOW_PCF5 : PCF 5x5 采样。返回到 WebGL 1.0 上的 SHADOW_PCF3。
      • settings.render.lightingCells : Vec3 : 沿包含灯光的空间细分为的每个世界空间轴的单元数。只有 bakeDir=true 的灯光才会用于生成主灯光方向。

📌 destroy()

在当前引擎帧更新结束时销毁应用程序并删除所有事件侦听器。但是，如果在引擎帧更新之外调用，则调用 destroy（）将立即销毁应用程序。

📌 drawLine(start : Vec3, end : Vec3, [color] : Color, [depthTest] : boolean, [layer] : Layer)
#

加载应用程序配置文件并应用应用程序属性并填写资产注册表。

参数列表

  • end : 线条的结束世界空间坐标。

📌 drawLineArrays(positions : number[], colors : number[], [depthTest] : boolean, [layer] : Layer)
#

渲染任意数量的离散线段。这些线不是由阵列中的每个后续点连接的。相反，它们是由两点指定的单独线段。

javascript
// 渲染2个离散线段
var points = [
  // Line 1
  0, 0, 0, 1, 0, 0,
  // Line 2
  1, 1, 0, 1, 1, 1,
];
var colors = [
  // Line 1
  1,
  0,
  0,
  1, // red
  0,
  1,
  0,
  1, // green
  // Line 2
  0,
  0,
  1,
  1, // blue
  1,
  1,
  1,
  1, // white
];
app.drawLineArrays(points, colors);

参数列表

  • positions : 要在其间绘制线的点数组。每个点由 3 个数字表示-x、y 和 z 坐标。
  • colors : 用于为线条着色的颜色数组。这必须与位置数组的长度相同。数组的长度也必须是 2 的倍数。
  • depthTest : 指定是否根据深度缓冲区对线进行深度测试。默认为 true。
  • layer : 要将线渲染到其中的层。默认为 LAYERID_IMMEDIATE。

📌 drawLines(positions : Vec3[], colors : Color[], [depthTest] : boolean, [layer] : Layer)
#

渲染任意数量的离散线段。这些线不是由阵列中的每个后续点连接的。相反，它们是由两点指定的单独线段。因此，提供的位置和颜色数组的长度必须相同，并且必须是 2 的倍数。每个线段末端的颜色将沿每条线的长度插值。

javascript
// 渲染单线，每个点具有唯一的颜色
var start = new pc.Vec3(0, 0, 0);
var end = new pc.Vec3(1, 0, 0);
app.drawLines([start, end], [pc.Color.RED, pc.Color.WHITE]);
JavaScript
// 渲染 2 个离散线段
var points = [
    // Line 1
    new pc.Vec3(0, 0, 0),
    new pc.Vec3(1, 0, 0),
    // Line 2
    new pc.Vec3(1, 1, 0),
    new pc.Vec3(1, 1, 1)
];
var colors = [
    // Line 1
    pc.Color.RED,
    pc.Color.YELLOW,
    // Line 2
    pc.Color.CYAN,
    pc.Color.BLUE
];
app.drawLines(points, colors);

参数列表

  • positions : 要在其间绘制线的点数组。数组的长度必须是 2 的倍数。
  • colors : 用于为线条着色的颜色数组。这必须与位置数组的长度相同。数组的长度也必须是 2 的倍数。
  • depthTest : 指定是否根据深度缓冲区对线进行深度测试。默认为 true。
  • layer : 要将线渲染到其中的层。默认为 LAYERID_IMMEDIATE。

📌 resizeCanvas([width] : number，[height] : number) : object
#

根据当前填充模式调整应用程序画布元素的大小。

  • 在 FILLMODE_KEEP_ASPECT 模式下，画布将增长到尽可能填充窗口，同时保持纵横比。
  • 在 FILLMODE_FILL_WINDOW 模式下，画布将简单地填充窗口，更改纵横比。
  • 在 FILLMODE_NONE 模式下，画布将始终与提供的大小匹配。

参数列表

  • width : 画布的宽度。仅在当前填充模式为 FILLMODE_NONE 时使用。
  • height : 画布的高度。仅在当前填充模式为 FILLMODE_NONE 时使用。

返回值

一个对象，包含计算出的用作宽度和高度的值。

📌 setCanvasFillMode(mode : string, [width] : number, [height] : number)

  • width : 画布的宽度（仅在模式为 FILLMODE_NONE 时使用）。

📌 setCanvasResolution(mode : string, [width] : number, [height] : number)
#

更改画布的分辨率，并设置调整窗口大小时画布的行为方式。

📌 start()
#

启动应用程序。此函数执行以下操作：

  1. 在名为“start”的应用程序上激发事件
  2. 调用层次结构中实体上所有组件的初始化
  3. 在名为“initialize”的应用程序上激发事件
  4. 为层次结构中实体上的所有组件调用 postInitialize
  5. 在名为“postinitialize”的应用程序上激发事件
  6. 开始执行应用程序的主循环

此函数由编辑器中的 PlayCanvas 应用程序在内部调用，但如果使用独立引擎，则需要自己调用 start。

javascript
app.start();

📌 update(dt : number)
#

更新应用程序。此函数将调用所有启用组件的更新函数，然后调用 postUpdate 函数。然后，它将更新所有连接的输入设备的当前状态。此函数在应用程序的主循环中内部调用，不需要显式调用。

参数列表

  • dt : 自上一帧以来的时间增量（秒）。

📌 updateCanvasSize()
#

更新 GraphicsDevice 画布大小以匹配文档页面上的画布大小。建议在画布大小更改时调用此函数（例如，在窗口大小调整和方向更改事件中），以便画布分辨率立即更新。
